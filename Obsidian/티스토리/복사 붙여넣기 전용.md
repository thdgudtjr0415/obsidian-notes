## 📝 복합쿼리 예제 (1) — 회원별 주문 집계

### 🔸 풀이 과정

#### ① FROM, WHERE절 — 날짜 범위 필터링

```sql
FROM TBL
WHERE 주문일시 BETWEEN '20240303' AND '20240315'
```

> 💡 **포인트:** `BETWEEN A AND B`는 A 이상 B 이하입니다.
> 20240302(A001 첫 주문)는 범위에서 제외됩니다.

**① 실행 결과**

| 주문번호 | 회원번호 | 주문일시 | 주문금액 |
|:---:|:---:|:---:|:---:|
| 002 | A001 | 20240310 | 3000 |
| 003 | A002 | 20240303 | 2000 |
| 004 | A002 | 20240311 | 2000 |
| 005 | A002 | 20240315 | 2000 |
| 006 | A003 | 20240304 | 1000 |
| 007 | A003 | 20240314 | 4000 |

---

#### ② GROUP BY, HAVING절 — 그룹핑 후 조건 필터

```sql
GROUP BY 회원번호
HAVING COUNT(*) >= 2
```

> 💡 **포인트:** `WHERE`는 그룹핑 전 행 필터링, `HAVING`은 그룹핑 후 그룹 필터링입니다.
> A001은 해당 기간 내 1건이라 제외, A002(3건), A003(2건)만 남습니다.

**② 실행 결과**

| 회원번호 | COUNT(*) |
|:---:|:---:|
| A002 | 3 |
| A003 | 2 |

---

#### ③ SELECT절 — 집계함수 적용

```sql
SELECT 회원번호,
       MAX(주문금액) AS 최고주문금액,
       SUM(주문금액) AS 주문금액합계
```

**③ 실행 결과**

| 회원번호 | 최고주문금액 | 주문금액합계 |
|:---:|:---:|:---:|
| A001 | 3000 | 3000 |
| A002 | 2000 | 6000 |
| A003 | 4000 | 5000 |

---

#### ④ ORDER BY절 — 주문금액합계 기준 내림차순

```sql
ORDER BY 주문금액합계 DESC;
```

**④ 최종 실행 결과**

| 회원번호 | 최고주문금액 | 주문금액합계 |
|:---:|:---:|:---:|
| A002 | 2000 | 6000 |
| A003 | 4000 | 5000 |
| A001 | 3000 | 3000 |

---

### ✅ 최종 완성 쿼리

```sql
SELECT 회원번호,
       MAX(주문금액) AS 최고주문금액,
       SUM(주문금액) AS 주문금액합계
FROM TBL
WHERE 주문일시 BETWEEN '20240303' AND '20240315'
GROUP BY 회원번호
HAVING COUNT(*) >= 2
ORDER BY 주문금액합계 DESC;
```

> 처음에 저는 HAVING 조건을 WHERE에 넣으려 했다가 오류가 났습니다.
> COUNT 같은 집계함수는 WHERE에서 쓸 수 없고 반드시 HAVING에 써야 한다는 걸
> 이 문제에서 확실히 잡았습니다.

### 🔸 풀이 과정

#### ① 서브쿼리 — 도서별 최초 대출시작일자 구하기

```sql
SELECT 도서ID, MIN(대출시작일자) AS 대출시작일자
FROM 대출
GROUP BY 도서ID
```

> 💡 **포인트:** `MIN()`으로 도서별 첫 번째 대출일만 추립니다.
> 이 결과를 D로 별명 지정해 메인쿼리에서 테이블처럼 활용합니다.

**① 서브쿼리 실행 결과 (D)**

| 도서ID | 대출시작일자 |
|:---:|:---:|
| 12001 | 2024-01-03 |
| 12002 | 2024-02-15 |
| 12003 | 2024-01-29 |

---

#### ② 메인쿼리 FROM, WHERE절 — 테이블 조인

```sql
FROM 대출 A, 도서 B, 출판사 C,
     (서브쿼리) D
WHERE A.대출시작일자 = D.대출시작일자
  AND B.도서ID = D.도서ID
  AND A.도서ID = B.도서ID
  AND A.출판사ID = C.출판사ID
```

> 💡 **포인트:** 서브쿼리 결과 D와 조인할 때
> **도서ID + 대출시작일자 두 조건 모두** 일치해야 "최초 대출" 행을 정확히 찾을 수 있습니다.
> 하나만 쓰면 CROSS JOIN이 발생할 수 있어요.

**② FROM, WHERE 조인 결과**

| 도서ID | 도서명 | 출판사명 | 대출시작일자 | 대출종료일자 |
|:---:|:---:|:---:|:---:|:---:|
| 12001 | SQL 개발자 정복 | 아이리포 | 2024-01-03 | 2024-01-13 |
| 12002 | 웹 애플리케이션 개발 | 나래출판 | 2024-02-15 | 2024-02-25 |
| 12003 | 기술사 길잡이 | 하늘도서 | 2024-01-29 | 2024-02-08 |

---

#### ③ 메인쿼리 SELECT, ORDER BY절

```sql
SELECT B.도서명, C.출판사명,
       TO_CHAR(A.대출시작일자, 'YYYY-MM-DD') AS 대출시작일자,
       TO_CHAR(A.대출종료일자, 'YYYY-MM-DD') AS 대출종료일자
ORDER BY B.도서명;
```

> 💡 **포인트:** `TO_CHAR(날짜, 'YYYY-MM-DD')`로 날짜를 보기 좋게 변환합니다.
> `ORDER BY B.도서명`으로 가나다순 정렬합니다.

**③ 최종 실행 결과**

| 도서명 | 출판사명 | 대출시작일자 | 대출종료일자 |
|:---:|:---:|:---:|:---:|
| SQL 개발자 정복 | 아이리포 | 2024-01-03 | 2024-01-13 |
| 기술사 길잡이 | 하늘도서 | 2024-01-29 | 2024-02-08 |
| 웹 애플리케이션 개발 | 나래출판 | 2024-02-15 | 2024-02-25 |

### ✅ 최종 완성 쿼리

```sql
SELECT B.도서명, C.출판사명,
       TO_CHAR(A.대출시작일자, 'YYYY-MM-DD') AS 대출시작일자,
       TO_CHAR(A.대출종료일자, 'YYYY-MM-DD') AS 대출종료일자
FROM 대출 A, 도서 B, 출판사 C,
     (SELECT 도서ID, MIN(대출시작일자) AS 대출시작일자
      FROM 대출
      GROUP BY 도서ID) D
WHERE A.대출시작일자 = D.대출시작일자
  AND B.도서ID = D.도서ID
  AND A.도서ID = B.도서ID
  AND A.출판사ID = C.출판사ID
ORDER BY B.도서명;
```

> 이 문제에서 가장 헷갈렸던 부분은 서브쿼리 결과를 어떻게 메인쿼리와 연결하느냐였습니다.
> 서브쿼리로 "도서별 최초 대출일"을 먼저 구하고, 그 결과를 하나의 테이블처럼 취급해서
> 조인한다는 흐름을 이해하고 나니 구조가 명확하게 보였습니다.

## 🔑 두 예제 핵심 포인트 정리

| 개념 | 핵심 |
|:---:|:---|
| WHERE vs HAVING | WHERE는 행 필터링, HAVING은 그룹 필터링 |
| 서브쿼리 | 복잡한 조건을 먼저 계산 후 메인쿼리에서 테이블처럼 활용 |
| MIN() + GROUP BY | 그룹별 첫 번째 값 추출할 때 조합 |
| TO_CHAR() | 날짜를 원하는 형식의 문자열로 변환 |
| BETWEEN A AND B | 범위 조건, A 이상 B 이하 |
